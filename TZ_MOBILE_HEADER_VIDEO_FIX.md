# Техническое задание: исправление воспроизведения видео в мобильном хедере

## 1. Цель

Обеспечить стабильное автоматическое воспроизведение фонового видео в первом экране сайта ESSENCE на мобильных устройствах, насколько это допускают ограничения браузера и операционной системы.

Если автоматический запуск запрещён настройками устройства или браузера, видео должно корректно запускаться после первого взаимодействия пользователя со страницей. Постер должен оставаться визуальным резервом до начала воспроизведения.

## 2. Область изменений

- Файл: `index.html`.
- Элемент видео: `#heroVideo`.
- Кнопка управления звуком: `#heroSoundBtn`.
- Мобильный ролик: `video/004-HEADER-9x16.mp4`.
- Десктопный ролик: `video/004-HEADER-16x9.mp4`.
- Мобильный постер: `img/posters/004-HEADER-9x16.jpg`.
- Десктопный постер: `img/posters/004-HEADER-16x9.jpg`.

Дизайн, тексты, положение элементов, высота первого экрана и внешний вид хедера менять не требуется.

## 3. Обнаруженная проблема

В текущей реализации мобильный источник видео подменяется JavaScript-кодом после первоначального разбора HTML:

1. изменяется `src` вложенного элемента `<source>`;
2. вызывается `heroVideo.load()`;
3. сразу вызывается `heroVideo.play()`;
4. ошибка запуска скрывается пустым обработчиком `catch`.

На мобильных браузерах, прежде всего Safari на iOS, вызов `load()` может сбросить состояние первоначального autoplay. Немедленный вызов `play()` выполняется до готовности нового источника и может быть отклонён. Текущий код не выполняет повторную попытку и не предусматривает запуск после взаимодействия пользователя.

Видеофайлы существуют. Контейнер MP4 и видеокодек H.264/AVC совместимы с основными мобильными браузерами. Основная зона исправления — логика выбора источника и запуска воспроизведения.

## 4. Требования к реализации

### 4.1. Сохранить обязательные атрибуты видео

Элемент `#heroVideo` должен сохранять следующие атрибуты:

```html
autoplay muted loop playsinline
```

Дополнительно перед любым программным запуском необходимо явно установить:

```js
heroVideo.muted = true;
heroVideo.defaultMuted = true;
heroVideo.playsInline = true;
heroVideo.setAttribute('muted', '');
heroVideo.setAttribute('playsinline', '');
```

Это должно выполняться до вызова `load()` и `play()`.

### 4.2. Надёжно назначать источник

Источник необходимо назначать самому элементу `<video>` через свойство `src`, а не изменять только `src` вложенного `<source>` после начала загрузки:

```js
heroVideo.src = isMobile
  ? 'video/004-HEADER-9x16.mp4'
  : 'video/004-HEADER-16x9.mp4';
```

После перехода на такой способ не должно оставаться конкурирующих источников, способных повторно заменить выбранный файл.

Мобильной считать ширину окна до 720 px включительно, в соответствии с существующим breakpoint сайта.

### 4.3. Синхронизировать постер и видео

Для каждого режима необходимо использовать соответствующую пару:

| Режим | Видео | Постер |
|---|---|---|
| Мобильный, до 720 px | `004-HEADER-9x16.mp4` | `004-HEADER-9x16.jpg` |
| Десктопный, более 720 px | `004-HEADER-16x9.mp4` | `004-HEADER-16x9.jpg` |

Постер должен отображаться до появления первого декодированного кадра и при невозможности загрузить видео.

### 4.4. Запускать видео после готовности источника

После назначения `src` требуется:

1. вызвать `heroVideo.load()`;
2. выполнить попытку `play()`;
3. если видео ещё не готово, повторить попытку однократно по событию `canplay` или `loadeddata`;
4. не создавать несколько одинаковых обработчиков и параллельных попыток запуска.

Рекомендуется оформить запуск в отдельной функции, возвращающей результат `heroVideo.play()`.

### 4.5. Не скрывать ошибку autoplay

Пустой обработчик вида:

```js
.catch(function(){})
```

необходимо удалить.

В режиме разработки допустим `console.warn` с типом ошибки. Ошибка autoplay не должна приводить к необработанному исключению или нарушать работу остальных скриптов страницы.

### 4.6. Добавить резервный запуск после взаимодействия

Если программный autoplay отклонён, необходимо зарегистрировать одноразовый резервный запуск после первого пользовательского взаимодействия:

- `pointerdown`;
- `touchstart` для старых мобильных браузеров;
- `keydown` для клавиатурной доступности.

После успешного запуска все резервные обработчики должны быть удалены.

Резервный запуск не должен включать звук. Видео остаётся беззвучным до отдельного нажатия на существующую кнопку звука.

### 4.7. Сохранить логику кнопки звука

Кнопка `#heroSoundBtn` должна продолжать:

- включать и выключать звук;
- корректно переключать класс `is-on`;
- обновлять локализованный `aria-label`;
- не останавливать видео при переключении звука.

Если видео ещё не запущено, нажатие на кнопку звука является пользовательским взаимодействием и должно также инициировать воспроизведение.

### 4.8. Обработка смены ориентации и размера

Минимальное обязательное поведение: правильный источник выбирается при первой загрузке страницы.

Желательное поведение: при фактическом переходе через breakpoint 720 px источник и постер переключаются без дублирования обработчиков. Обычные изменения высоты окна и появление/скрытие адресной панели мобильного браузера не должны перезагружать видео.

### 4.9. Учитывать системные ограничения

На iOS в режиме энергосбережения, при включённом ограничении движения или при других системных запретах браузер может не разрешить autoplay даже для беззвучного видео.

В таком случае:

- сайт не должен показывать пустой или чёрный экран;
- должен оставаться корректный постер;
- видео должно запускаться после первого разрешённого взаимодействия, если браузер это допускает;
- остальные элементы первого экрана должны работать без ошибок.

Не следует принудительно отключать видео только на основании `prefers-reduced-motion`, если это не согласовано отдельно. При необходимости поддержки этой настройки оставить постер и не выполнять autoplay.

## 5. Рекомендуемая структура логики

Реализацию рекомендуется разделить на небольшие функции:

- определение мобильного режима;
- установка соответствующих `src` и `poster`;
- безопасная попытка воспроизведения;
- подключение и удаление резервных обработчиков;
- переключение звука.

Нельзя создавать таймер с постоянными повторными вызовами `play()`. Нельзя бесконечно перезагружать видео при ошибке.

## 6. Что не менять

- CSS-композицию первого экрана.
- Соотношение сторон и `object-fit: cover`.
- Контент, переводы и типографику.
- Навигацию и поведение меню.
- Файлы основного раздела `Le Film`.
- Имена существующих видео и постеров без необходимости.
- Внешний вид кнопки звука.

## 7. Проверка после внесения изменений

### 7.1. Функциональная проверка

Для `#heroVideo` проверить:

- загружен правильный файл в зависимости от ширины экрана;
- `muted === true` до первого нажатия кнопки звука;
- `playsInline === true`;
- видео не открывается в отдельном полноэкранном плеере на iPhone;
- `currentTime` увеличивается после запуска;
- ролик повторяется после завершения;
- при ошибке запуска остаётся постер;
- первое взаимодействие запускает видео, если autoplay был отклонён;
- в консоли нет необработанных ошибок.

### 7.2. Матрица устройств и браузеров

Минимально проверить:

| Платформа | Браузер | Ориентация |
|---|---|---|
| iPhone, актуальная iOS | Safari | вертикальная |
| iPhone, актуальная iOS | Safari | горизонтальная |
| Android | Chrome | вертикальная |
| Android | Chrome | горизонтальная |
| macOS | Safari | десктопная |
| Windows/macOS | Chrome | десктопная |

Отдельно проверить iPhone с включённым режимом энергосбережения: допустимо отсутствие autoplay, но постер и резервный запуск должны работать корректно.

### 7.3. Проверка сети

Проверить при обычном соединении и при эмуляции медленной мобильной сети:

- первый экран появляется без пустой области;
- постер не исчезает раньше готовности видео;
- видео не загружается одновременно в мобильной и десктопной версиях;
- сервер возвращает для MP4 корректный `Content-Type: video/mp4`;
- сервер поддерживает byte-range запросы и возвращает `206 Partial Content` при запросе диапазона.

## 8. Критерии приёмки

Работа считается выполненной, если:

1. На поддерживающих autoplay мобильных устройствах видео в хедере автоматически запускается без звука и воспроизводится внутри страницы.
2. На ширине до 720 px загружается вертикальный ролик и соответствующий постер.
3. На ширине более 720 px загружается горизонтальный ролик и соответствующий постер.
4. При блокировке autoplay пользователь видит постер, а первое разрешённое взаимодействие запускает ролик.
5. Ошибки `play()` не скрываются полностью и не ломают остальные скрипты.
6. Кнопка управления звуком сохраняет текущее поведение и доступность.
7. Не происходит одновременная загрузка двух версий ролика.
8. Визуальная композиция хедера не изменяется.

## 9. Результат работ

Исполнитель должен предоставить:

- обновлённый `index.html`;
- краткое описание внесённых изменений;
- перечень проверенных устройств и браузеров;
- подтверждение отсутствия необработанных ошибок в консоли;
- при наличии ограничений конкретного устройства — описание воспроизводимого сценария и поведения fallback.
